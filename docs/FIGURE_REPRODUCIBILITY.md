# Figure Reproducibility

README figures are generated by the analysis notebooks and written to stable paths under `assets/`. Rerunning the relevant notebooks refreshes the files that the README displays. The README emphasizes the clinical result flow rather than validation visuals.

## Figure Exports

| README figure | Exporting notebook | Output path |
| --- | --- | --- |
| Unadjusted in-hospital mortality by early RAAS exposure | `notebooks/04a_unadjusted_outcomes.ipynb` | `assets/unadjusted_mortality_by_raas.png` |
| Age-specific adjusted absolute risk difference | `notebooks/04b_multivariable_outcomes.ipynb` | `assets/absolute_risk_difference_plot.png` |

The adjusted odds ratio figure (`assets/adjusted_odds_ratio_plot.png`) and SAS-Python validation summary (`assets/sas_python_validation_summary.png`) remain reproducible notebook exports, but they are not primary README visuals.

## Refresh Workflow

1. Run the relevant notebook from the repository root or from the `notebooks/` directory.
2. The notebook resolves the project root, creates `assets/` if needed, and saves the figure using the stable filename above.
3. Because the README references those stable paths, the README visuals update after the notebook is rerun and the refreshed assets are committed.

## Data Protection

The figure files contain only aggregate, presentation-level summaries. No patient-level records, local analysis extracts, credentials, SAS input files, or protected health information are stored in `assets/`.

## Interpretation Boundary

The README figures summarize observational associations. They should not be interpreted as causal treatment-effect evidence. Residual confounding may remain despite multivariable adjustment and sensitivity analyses. SAS-Python validation remains a reproducibility check based on aggregate output comparison.
